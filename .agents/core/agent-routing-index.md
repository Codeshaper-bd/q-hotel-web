# Agent Routing Index

## Mission

Help future AI work choose the right instruction files before making changes.

## Always Read

For any code or architecture change:

1. `AGENTS.md`
2. `.agents/core/user-approval-gate.md`
3. `.agents/core/agent-routing-index.md`
4. `.agents/core/project-conventions-guardian.md`

For styling changes, also read:

5. `.agents/frontend/css-implementation-guardian.md`

Before final handoff for meaningful changes, read:

6. `.agents/quality/release-quality-gate.md`

## Task Routing

Use these files by task type:

- Architecture or page/module structure: `.agents/core/premium-frontend-architect.md`
- Creating/updating agent or skill files: `.agents/core/skill-authoring-governor.md`
- Context handoff, resume notes, or architecture decisions: `.agents/core/project-memory-guide.md`
- Routes, nav, header/footer IA: `.agents/content-seo/site-information-architecture.md`
- Page sections such as hero/features/gallery/CTA: `.agents/frontend/section-composition-architect.md`
- Hotel business logic and domain models: `.agents/backend/business-logic-domain-modeler.md`
- Naming, folders, variables, imports: `.agents/core/project-conventions-guardian.md`
- Component props, emits, slots, variants: `.agents/frontend/component-api-designer.md`
- TypeScript, composables, utilities: `.agents/quality/typescript-code-quality.md`
- Tailwind/CSS implementation: `.agents/frontend/css-implementation-guardian.md`
- Tokens and visual system: `.agents/frontend/design-system-guardian.md`
- GSAP, animation, WebGL: `.agents/frontend/motion-webgl-engineer.md`
- Performance, bundles, lazy loading: `.agents/quality/performance-budget-auditor.md`
- Dependencies and upgrades: `.agents/workflow/dependency-governor.md`
- Images, video, models, fonts: `.agents/content-seo/media-asset-optimizer.md`
- SEO metadata, sitemap, schema: `.agents/content-seo/technical-seo-structured-data.md`
- Accessibility and semantic HTML: `.agents/content-seo/seo-accessibility-auditor.md`
- API, CMS, server routes: `.agents/backend/api-integration-architect.md`
- Contact, inquiry, or reservation forms: `.agents/backend/forms-contact-flow.md`
- Runtime config and environment variables: `.agents/backend/environment-config-governor.md`
- Security, privacy, forms, storage: `.agents/backend/security-privacy-guardian.md`
- Analytics, tracking, cookie consent: `.agents/backend/analytics-consent-governor.md`
- Content models and CMS readiness: `.agents/content-seo/content-modeler.md`

- Deployment, hosting, build output: `.agents/workflow/deployment-guide.md`
- Git, commits, PR hygiene: `.agents/workflow/git-workflow-keeper.md`

## Conflict Resolution

When files conflict, use this priority:

1. Security and privacy
2. Accessibility
3. Performance/Core Web Vitals
4. SEO crawlability
5. Maintainability/type safety
6. Visual polish and motion

If a user request conflicts with the project standard, explain the tradeoff and implement the closest standards-compliant option.
